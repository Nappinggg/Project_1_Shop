
1. Опис програми
Це консольний застосунок C#, написаний для платформи .NET 8. Програма називається ShopApp і є навчальним (або демонстраційним) проєктом.
Її мета — продемонструвати дві основні концепції:
Основи Об'єктно-Орієнтованого Програмування (ООП):
Програма визначає три окремі класи, що описують сутності "магазину":
Product (Товар): Має ID, Назву, Ціну та Наявність.
Customer (Клієнт): Має Повне ім'я, Вік, Баланс та Статус (постійний/новий).
Order (Замовлення): Має ID замовлення, Ім'я клієнта, Загальну суму та Статус (виконано/в процесі).

Базові Колекції C#:
У головному файлі Program.cs демонструється, як створювати та використовувати найпоширеніші структури даних: масив (int[]), список (List), чергу (Queue), стек (Stack), словник (Dictionary) та множину (HashSet).

2. Робота програми (Як вона виконується)
Виконання відбувається покроково у файлі Program.cs всередині методу Main:
Налаштування: Вмикається кодування UTF-8 для коректного відображення українських символів в консолі.
Створення об'єктів (ООП):
Створюється екземпляр Product (товар "Амазон Кіндел").
Викликається метод PrintProduct, який форматовано виводить в консоль всю інформацію про цей товар.
Створюється екземпляр Customer (клієнт "Вася Пупкін").
Викликається метод PrintCustomer для виведення інформації про клієнта.
Створюється екземпляр Order (нове замовлення), який логічно пов'язує "Васю Пупкіна" з "Амазон Кіндел".
Викликається метод PrintOrder для виведення деталей замовлення.

Демонстрація Колекцій:
Масив: Створюється масив int[] prices і виводиться в один рядок.
Список: Створюється List<string> clients, до нього додається нове ім'я ("Аполінарія"), і список виводиться в рядок.
Черга: Створюється Queue<string>. Демонструється принцип "Перший прийшов - перший пішов" (FIFO): програма показує, хто перший у черзі (Peek), потім "обслуговує" його (Dequeue), і показує, хто став новим першим.
Стек: Створюється Stack<string>. Демонструється принцип "Останній прийшов - перший пішов" (LIFO): програма показує останню додану дію ("Зробив замовлення") за допомогою Peek.
Словник: Створюється Dictionary<int, string> (база товарів), і виводяться всі її пари "ключ -> значення".
Множина: Створюється HashSet<string> (унікальні замовлення) і виводяться його елементи.

3. Мої коментарі та аналіз (Як розробника)
   Створив цей проект не для складної бізнес логіки, а як пісочницю. Моєю головною метою було в одном місці продемонструвати синтаксис і базові концепції: ось так виглядають класи (ООП), а ось так найпоширеніші колекціІ С#

   Виніс Product, Order та Customer в окремі файли. Це стандартна практика, яка робить код чистішим. Я не хотів звалити все в купу в Program.cs, щоб було легше читати кожен клас окремо.

   Я розумію, що файл Program.cs зараз робить забагато — він містить і логіку виконання (Main), і методи для відображення (PrintProduct тощо).  
     Якби це був більший проєкт, я б так не робив. Я б, скоріше за все, або створив окремий клас-хелпер для виводу в консоль, або ж (що правильніше) перевизначив (override) метод ToString() у кожному класі (Product, Customer...), щоб вони самі відповідали за те, як себе представляти у вигляді рядка.
     Але для цієї демонстрації, де мета — швидко показати результат, зібрати все в одному місці було простіше.
   
   Використав стандартний шаблон .NET 8. Саме тому файл .csproj такий маленький — він автоматично підключає ImplicitUsings (не треба писати using System... скрізь) і Nullable (допомагає уникати null помилок), що просто робить код сучаснішим і коротшим.

   Спочатку показую, як об'єкти класів працюють разом (створюємо товар і клієнта, потім "зв'язуємо" їх у замовленні). І лише після цього, окремим блоком, я проходжуся по різних структурах даних (масив, список, стек...). Я вважаю, що таке розділення допомагає краще засвоїти кожну тему окремо.
